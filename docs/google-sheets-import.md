# Google Sheets — импорт данных

## Статус: реализовано (базовая версия)

Страница `/sheets-import`, сервисный аккаунт `sheets-import@liquid-journal-454008-b1.iam.gserviceaccount.com`,
ключ в `credentials.json` (в `.gitignore`, не коммитится).

## Как это работает сейчас

1. Пользователь вставляет ссылку на Google Sheet → жмёт **"Sekmeleri Getir"**.
   - `POST /api/sheets-import/tabs` — извлекает `SPREADSHEET_ID` из ссылки, через Sheets API получает список
     вкладок (`internal/googlesheets/client.go: ListTabs`), сохраняет ссылку в таблицу `sheet_links`
     (upsert по `spreadsheet_id`, чтобы при повторном визите не вставлять ссылку заново — они показываются
     списком "Kayıtlı bağlantılar" на странице).
2. Список вкладок показывается кликабельными карточками (не `<select>`).
3. Клик по вкладке:
   - Если название вкладки **≠ "Turlar"** — обычный режим: `POST /api/sheets-import/preview` возвращает
     ВСЕ строки как есть (без угадывания заголовков — на разных вкладках первая строка может быть служебной:
     кнопки, пустые строки и т.д.), колонки подписаны как в самом Google Sheets (A, B, C...).
   - Если название вкладки **== "Turlar"** — специальный режим (см. ниже).

## Вкладка "Turlar" — специфичная бизнес-логика

Эта вкладка есть всегда в таблицах пользователя, в неё только добавляют новые туры (кнопки "Tur Ekle" /
"Tur İçin Sayfa Oluştur" прямо в самом Google Sheet). Структура нестандартная: сверху несколько строк
служебные (лейблы, кнопки, инпуты), реальная строка заголовков — это первая строка, где есть ячейка
**"Tur Kodu"** (ищется по всей вкладке, а не по фиксированному номеру строки, см.
`internal/handler/sheets_import_turlar.go: findTurlarHeader`).

Колонки определяются по названию заголовка (регистронезависимо, с азербайджанскими/турецкими вариантами):
- `Tur Kodu` → код тура
- содержит `başlan`/`baslan` → дата начала
- содержит `bitiş`/`bitis` → дата окончания
- содержит `otaq`/`oda` → тип комнаты (QDR/TRP/DBL/SNG/DBL-TWN/TWN...)
- все остальные непустые колонки после этого → категории тура (ECO-Ekonomik, ECY-Ekonomik Yürüme,
  SLV-Gümüş, GLD-Altın, PLT-Platin...), значение в ячейке — цена этой категории для этой строки/комнаты

`POST /api/sheets-import/turlar-candidates` группирует строки по `Tur Kodu` и возвращает по каждому туру
список "items" (комбинации room_code + category_name + price), а также флаг `exists` — **сравнение
только по `Tur Kodu`** против `tours.code` в нашей БД (решение пользователя: не сравнивать по
комнате/категории, только по коду тура).

### Модель данных: почему это важно

`tours.room_id` и `tours.tour_category_id` — обычные `NOT NULL` FK (один-ко-многим, не many-to-many).
Значит один Tur Kodu с 6 типами комнат → 6 отдельных строк в `tours` с одинаковым `code`.

`rooms` раньше не имел текстового кода (только `price` + `beds_count`) — добавлено поле
`rooms.code` (migration `021_add_code_to_rooms.sql`) специально для сопоставления с "Otaq Tipi"
из Google Sheet.

### Импорт — сопоставление и подтверждение (решение пользователя)

При импорте тура (кнопка "İçe Aktar" на конкретном туре или "Tümünü İçe Aktar" сверху — идёт
последовательно, по одному туру за раз):

1. Для каждого item (room_code + category_name + price):
   - Ищем `Room` по `code` (регистронезависимо). Если нет — **спрашиваем подтверждение** (`confirm()`
     в браузере) на создание новой комнаты с ценой из этой строки и угаданным `beds_count`
     (QDR=4, TRP=3, SNG=1, остальное=2 — эвристика, правится потом вручную на странице "Odalar").
   - Ищем `TourCategory` по `name` (регистронезависимо). Если нет — так же спрашиваем подтверждение.
   - Если пользователь отменил создание — эта конкретная строка (item) пропускается, но
     остальные items тура продолжают обрабатываться.
2. После резолва room_id/category_id — создаём `Tour` (`POST /api/tours`).
3. Только после этого код помечается как "Mövcuddur" (существует) в UI.

Это значит: у одного тура (Tur Kodu) при первом импорте может быть несколько confirm-диалогов подряд
(один на каждую новую комнату + один на каждую новую категорию), но категория обычно переиспользуется
между строками одного тура (создаётся один раз).
